# Пояснительная записка: VK Quiz

## 1. Цель проекта

VK Quiz — адаптивное веб-приложение для проведения интерактивных квизов в реальном времени. Организаторы создают квизы, публикуют их в каталоге или делают приватными, запускают live-комнаты, а участники подключаются по коду и отвечают на вопросы синхронно.

## 2. Средства разработки

Для frontend выбран Next.js + React + TypeScript. Для realtime-части используется отдельный Node.js-сервер с Socket.IO. Данные хранятся в PostgreSQL, работа с базой реализована через Prisma ORM. Для дизайна используются Tailwind CSS, кастомные компоненты и адаптивная дизайн-система.

## 3. Авторизация и подтверждение email

В проекте реализованы регистрация, вход и выход пользователя. После регистрации создаётся токен подтверждения email. Письмо может отправляться в локальный Mailpit, через реальный SMTP или через Resend. До подтверждения email пользователь не может войти в аккаунт.

## 4. Каталог квизов

Квизы делятся на публичные и приватные. Публичные отображаются в каталоге `/catalog`. Приватные скрыты и доступны только по коду доступа. Это позволяет поддерживать как открытые учебные квизы, так и закрытые мероприятия.

## 5. Работа с изображениями

Организатор может создать и полноценно редактировать квиз: название, описание, категорию, статус, публичность, приватный код, обложку, порядок вопросов, текст вопросов, картинки, время, баллы, варианты и правильные ответы. Также можно опционально добавить обложку квиза и изображение к любому вопросу. Поддерживается загрузка файла и вставка ссылки на изображение. Если обложка не задана, система автоматически показывает фирменную fallback-обложку.

## 6. Realtime-функциональность

Live-прохождение реализовано через WebSocket/Socket.IO. Участники подключаются к комнате по коду, ведущий запускает вопрос, ответы собираются в режиме реального времени, после чего отображается лидерборд. Дополнительно добавлен solo-режим: пользователь может пройти опубликованный квиз самостоятельно без организатора.

## 7. Дизайн

Интерфейс поддерживает светлую и тёмную темы с анимированным переключателем. Для персонализации пользователя добавлен 2D/псевдо-3D маскот, который используется в кабинете и профиле. Стандартные emoji заменены на собственные SVG-иконки, а логотип выполнен как Brain Pulse mark.

## 8. Итог

Получен MVP, соответствующий требованиям: регистрация, роли, каталог, приватные квизы, solo-прохождение без организатора, полноценный редактор квизов и вопросов, изображения, WebSocket-комнаты, лидерборд, история и современный адаптивный интерфейс.


## 9. Система баллов

Система подсчёта баллов стала дифференцированной. В вопросах с одиночным выбором пользователь получает полный балл только за правильный вариант, иначе 0. В вопросах с множественным выбором каждый правильно выбранный вариант даёт равную долю баллов, а каждый лишний выбранный вариант вычитает такую же долю. Итог не может быть меньше 0. Ответ после окончания таймера в live-режиме даёт 0 баллов.

### Дополнительные UX-функции

В каталоге реализованы поиск по названию/описанию и фильтрация по категории. После самостоятельного прохождения пользователь может поставить оценку от 1 до 5. Каталог сортирует квизы по средней оценке и количеству прохождений, что помогает показывать наиболее качественные и популярные квизы первыми. В редакторе квиза добавлены предупреждения о несохранённых изменениях и возможность сохранить черновик перед выходом.

### UX-доработки темы и мобильного интерфейса

В финальной версии розовая тема оформлена как полноценная цветовая схема: логотип, кнопки, статусы, предупреждения, бейджи и интерактивные элементы используют разные оттенки розового. На фоне добавлена лёгкая декоративная анимация сердечек. Для мобильной версии сокращена кнопка переключения роли до иконки, чтобы верхняя панель не перегружалась. Также добавлен переключатель звуков интерфейса с сохранением настройки в браузере.


## Последние UX-правки

Добавлена mobile-first навигация, компактная шапка, нижнее меню, примерное время прохождения квиза, обновлённые карточки каталога и более стабильная система звуков ответов.

### Обновление логики оценки и прохождения

В финальной версии добавлено ограничение: один авторизованный пользователь может оставить только одну оценку для одного квиза. Это предотвращает искусственное накручивание рейтинга. Активный экран прохождения упрощён: во время ответа пользователь видит только вопрос и варианты ответа, без названия квиза и промежуточных баллов. Баллы и результат отображаются по окончании прохождения.
