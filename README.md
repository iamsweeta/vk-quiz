# VK Quiz

Адаптивная realtime-платформа для квизов: каталог публичных квизов, приватные квизы по коду, solo-прохождение без организатора, авторизация, подтверждение email, live-комнаты, WebSocket и лидерборд.

## Что внутри

```txt
vk-quiz/
├── apps/
│   ├── web/                 # Next.js frontend + API routes
│   └── realtime-server/     # Socket.IO/Fastify realtime backend
├── packages/
│   ├── db/                  # Prisma schema + seed
│   ├── shared/              # общие события и scoring
│   └── ui/                  # заготовка UI-пакета
├── docs/
│   └── ARCHITECTURE.md
├── docker-compose.yml       # PostgreSQL + Mailpit
├── .env.example
└── APPLICATION_NOTE.md
```

## Быстрый запуск на Windows / PowerShell

```powershell
cd D:\vk-quiz
Copy-Item .env.example .env
npm.cmd install
docker compose up -d
npm.cmd run db:generate
npm.cmd run db:migrate
npm.cmd run db:seed
npm.cmd run dev
```

Открыть приложение:

```txt
http://localhost:3000
```

Realtime health-check:

```txt
http://localhost:4000/health
```

Mailpit для dev-писем:

```txt
http://localhost:8025
```

## Email-подтверждение

По умолчанию включён локальный Mailpit:

```env
EMAIL_PROVIDER="mailpit"
MAILPIT_HOST="localhost"
MAILPIT_PORT="1025"
```

Для реальной почты через SMTP:

```env
EMAIL_PROVIDER="smtp"
SMTP_HOST="smtp.gmail.com"
SMTP_PORT="465"
SMTP_SECURE="true"
SMTP_USER="your-email@gmail.com"
SMTP_PASSWORD="your-app-password"
EMAIL_FROM="VK Quiz <your-email@gmail.com>"
APP_URL="http://localhost:3000"
```

Для Gmail нужен App Password, а не обычный пароль аккаунта.

Для Resend:

```env
EMAIL_PROVIDER="resend"
RESEND_API_KEY="re_xxxxxxxxx"
EMAIL_FROM="VK Quiz <noreply@your-domain.com>"
APP_URL="https://your-domain.com"
```

## Картинки

В конструкторе квиза можно опционально добавить:

- обложку квиза;
- изображение к каждому вопросу.

Поддерживается загрузка файла в `apps/web/public/uploads` и ручная вставка URL изображения. Если организатор не добавил обложку, интерфейс автоматически показывает фирменную fallback-обложку с логотипом VK Quiz и названием квиза.


## Редактирование квизов

В кабинете организатора доступен полноценный редактор квиза:

- название, описание и категория;
- статус: черновик, опубликован, архив;
- публичный или приватный доступ;
- accessCode для приватного квиза;
- обложка квиза или автоматическая fallback-обложка;
- добавление, удаление и изменение порядка вопросов;
- текст вопроса, картинка, время и баллы;
- один или несколько правильных ответов;
- добавление и удаление вариантов ответа;
- опциональная картинка для варианта ответа.


## Solo-прохождение и начисление баллов

Квиз можно пройти двумя способами:

```txt
Solo mode -> участник проходит квиз сам, без организатора и live-комнаты
Live mode -> организатор показывает вопросы в комнате через WebSocket
```

Правило scoring одинаковое и прозрачное:

```txt
правильный ответ -> полная стоимость вопроса
неправильный ответ -> 0 баллов
multiple choice -> баллы только если выбран весь правильный набор без лишних вариантов
ответ после таймера в live mode -> 0 баллов
```

## Публичные и приватные квизы

```txt
PUBLIC  -> видны в /catalog
PRIVATE -> скрыты из каталога и открываются по accessCode
Room.code -> отдельный код для подключения к live-комнате
```

Демо-коды:

```txt
Публичная live-комната: QZ-4821
Историческая live-комната: HIST-2026
Приватный accessCode: AI-PRIVATE-777
Приватная live-комната: AI-777
```

## Если порт 5432 занят

Проект использует PostgreSQL на локальном порту `5433`, чтобы не конфликтовать с другими базами:

```env
DATABASE_URL="postgresql://quizpulse:quizpulse@localhost:5433/quizpulse?schema=public"
```

Docker пробрасывает:

```txt
5433 -> 5432 внутри контейнера
```

## Обновление: каталог, роли, рейтинг и UX-предупреждения

В этой версии добавлены:

- кнопка переключения роли рядом с основными пунктами навигации;
- предупреждение при уходе со страницы редактирования квиза с несохранёнными изменениями;
- предупреждение при выходе из solo-квиза до завершения;
- 15 базовых категорий и вариант «Другое»;
- поиск по названию и описанию в каталоге;
- фильтр каталога по категории;
- рейтинг квиза по 5-бальной шкале после solo-прохождения;
- счётчик прохождений;
- сортировка каталога: сначала высокая оценка, затем количество прохождений.

## Последние UX-доработки

- Розовая тема теперь полностью розовая: логотип, акценты, кнопки, предупреждения и статусы используют розовые оттенки.
- В розовой теме на фоне появляются лёгкие анимированные сердечки.
- На мобильной версии переключение роли вынесено в компактную кнопку с иконкой стрелок по кругу.
- Добавлен переключатель звуков интерфейса. Настройка сохраняется в браузере (`quizpulse-sound`).


## Последние UX-правки

Добавлена mobile-first навигация, компактная шапка, нижнее меню, примерное время прохождения квиза, обновлённые карточки каталога и более стабильная система звуков ответов.

## Последние UX-правки

- Один пользователь может оценить один квиз только один раз. Повторная оценка блокируется на уровне API и базы данных через уникальную пару `quizId + userId`.
- Во время solo/live-прохождения интерфейс больше не показывает название квиза и текущие баллы: остаются только номер вопроса, сам вопрос, варианты ответов и таймер. Итоговые баллы показываются после завершения квиза.

Если база уже была создана в старой версии, после обновления схемы можно выполнить:

```powershell
npx.cmd prisma migrate reset --schema packages/db/prisma/schema.prisma
npm.cmd run db:seed
```
